# SignOra v0.7

## Reference pose ensemble

Validators fuse multiple pose backends into a single reference (anti-MediaPipe-circular):

| Backend | Module | Required |
|---------|--------|----------|
| MediaPipe Holistic | `pose/backends/mediapipe_backend.py` | `pip install signora[pose]` |
| Optical flow pseudo-landmarks | `pose/backends/optical_flow.py` | opencv (default) |
| DWPose ONNX | `pose/backends/dwpose_onnx.py` | `models/dwpose.onnx` + onnxruntime |

```python
from signora.pose.ensemble import ReferenceEnsemble
ref = ReferenceEnsemble().extract_from_path("clip.mp4", "clip_s2_hello", stage=2)
```

## Commit-reveal weights

`signora/validation/weight_committer.py` uses, in order:

1. **CRv3 timelock** — `commit_timelocked_mechanism_weights` when subnet has commit-reveal enabled
2. **Commit-reveal** — `commit_mechanism_weights` + deferred `reveal_mechanism_weights`
3. **Direct** — `set_mechanism_weights` fallback

Pending salt commits persist in `data/weight_commits.json`.

## Challenge server

Known-plaintext capture with encrypted script storage:

```bash
export SIGNORA_CHALLENGE_SECRET=your-secret
pip install signora[server]
python scripts/run_challenge_server.py --port 8787
```

| Endpoint | Purpose |
|----------|---------|
| `POST /ingest` | Upload video + script (script encrypted, commit stored) |
| `GET /challenges` | List unrevealed clips for validators |
| `POST /reveal` | Decrypt script after DBCP window |
| `GET /manifest` | Audit commits without plaintext |

## Train Comp B
